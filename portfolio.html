Cardiac AR

Project : AR Mobile Apps

Software : Unity, Vuforia

Client : Dr Izzah, UPM

Preview : Project File

After graduating, I landed a freelance job developing a mobile augmented reality app for learning heart anatomy. My client was from the Faculty of Medicine at UPM, and the project was funded by a research grant. The goal was to create a powerful teaching tool for classrooms.

By harnessing the power of AR, the app enabled students to visualize how the heart works, understand blood flow, learn about the heart's conduction system, and explore the different stages of the cardiac cycle through engaging interactivity. It encompassed comprehensive notes, intriguing trivia, and even featured real MRI scans of heart anatomy.

The development process followed a meticulous path, starting with planning and analysis, followed by UI/UX design, app development, testing, and finally, deployment. I utilized Unity as the primary development software, integrating Vuforia for seamless AR implementation. Figma proved invaluable for the design process. I actively participated in testing and received positive feedback from students, which was incredibly rewarding. In fact, the app's success led me to be awarded a gold medal at the prestigious PiCTL event held at UPM.

Inspired by the project's impact, I decided to pursue a Master's degree, focusing on researching stereoscopic augmented reality in the context of learning heart anatomy. This academic journey allows me to delve deeper into the potential of AR technology and its applications in medical education, with the ultimate aim of revolutionizing the way students understand and engage with complex anatomical concepts.

Tele-Marz

Project : Mobile Application

Software : Unity 3d, Figma, Blender

Client : Dr Marzni, UPM

Preview : Google Playstore

As a dedicated developer, I collaborated with a client to create a mobile app for secondary school students undergoing language transitioning. The app seamlessly integrates with an animation book, serving as a valuable learning tool.

For this project, I utilized a range of powerful development tools. Unity, coupled with C#, enabled me to create functional mobile app features. Blender came in handy for crafting captivating 3D models featuring a gamified style Agong of Malaysia. PAC System

During my one-year tenure as a research assistant, I worked on an exciting project involving the creation of an augmented reality (AR) mobile application integrated with a web-based PACS System. The PACS System, designed for clinics and hospitals, aimed to store MRI scan image data, allowing doctors to view and utilize a marker for immersive 3D model visualization through the AR mobile application. This feature enhanced the doctor's ability to explain medical concepts in a more engaging and interactive manner. The web-based system utilized a cloud server with Linux/Ubuntu OS on a DigitalOcean droplet. I was responsible for setting up the Linux environment, configuring Nginx as the web server, utilizing Node.js for read and write operations, and implementing MongoDB for the database. Additionally, I documented the project thoroughly.

The PAC System mobile application, powered by Unity Engine and Vuforia, facilitated the visualization of anatomical structures for patients. By scanning a marker on the PAC System, the AR mobile application displayed a model with the option to draw on top of it, offering a stereoscopic view and enabling interaction through head gaze. While I did not develop the web-based system itself, I successfully established the Linux-based cloud server on DigitalOcean, integrating key components such as Nginx, Node.js, and MongoDB. I also connected the server to the domain and seamlessly incorporated Orthanc API to enhance the PAC System's functionality. Notably, I implemented a delete button, enabling the removal of images from the PAC System.

This project highlighted my contributions to both the AR mobile application and the server setup, underscoring my skills in creating immersive and practical solutions for medical imaging.

Stereoscopi Augmented Reality approch for Learning Heart Anatomy

Project : Stereoscopic Augmented Reality

Software : Unity 3D, IBM, Manomotion

Subject : Multimedia Computing / HCI

Preview : Project File

In my Master's research, I focused on utilizing a stereoscopic augmented reality approach to facilitate the learning of heart anatomy. By leveraging Head-Mounted Display (HMD) technology, I transformed 3D models of the heart and employed various interaction methods for enhanced engagement. These included virtual buttons using Vuforia, head gaze implementation in Unity, voice recognition with the IBM SDK, and gesture interaction using the Manomotion SDK.

The study aimed to address the challenge of visualizing and concretizing abstract concepts through augmented reality, which can significantly increase students' interest and motivation in learning. However, wearable augmented reality presents usability challenges, particularly in self-learning scenarios. Wearable AR, with its dynamic and complex 3D models, has the potential to provide context-aware assistance and deliver multimodal cues like animation, graphics, text, video, voice, and tactile feedback through wearable computing devices.

To overcome the challenges, it was crucial to identify suitable methods of interaction with the virtual 3D model. The familiarity and universality of user inputs for mobile applications served as a basis. For example, pinching fingers to zoom or double-tapping to highlight objects are well-known gestures. However, in the medical field, specific challenges arise when using wearable augmented reality, especially with regards to user input and creating suitable surroundings.

Through initial experiments, I examined four different interaction methods: virtual buttons, head gaze interaction, voice recognition, and gesture interaction. The results indicated that user preferences varied, but head gaze interaction showed promising potential for interacting with the 3D model via wearable devices. This finding contributes to identifying suitable interaction methods for practical and usable stereoscopic augmented reality applications in the medical field.
